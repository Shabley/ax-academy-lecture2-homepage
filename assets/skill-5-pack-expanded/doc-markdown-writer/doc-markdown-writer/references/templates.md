# 문서 유형별 마크다운 템플릿

각 템플릿은 골격이다. 입력 내용에 맞춰 섹션을 더하거나 빼고, 표/불릿/체크박스로 채운다.
모든 템플릿 공통: **맨 위에 한 줄 요약 또는 `> 핵심 요약` blockquote를 둔다.**

---

## 1. 보고서 (Report)

```markdown
# [보고서 제목]

> **핵심 요약:** [3줄 이내. 무엇을, 왜, 결과/제언이 무엇인지.]

**작성일** · **작성자/소속** · **대상**

---

## 1. 배경 및 목적
[왜 이 보고가 필요한가. 1~2문단.]

## 2. 현황 / 분석
[데이터·사실. 가능하면 표로.]

| 항목 | 내용 | 비고 |
|------|------|------|
|  |  |  |

## 3. 주요 결과 / 발견
- 핵심 결과 1
- 핵심 결과 2

## 4. 결론 및 제언
1. [실행 가능한 제언]
2. [실행 가능한 제언]

## 5. 향후 계획 / 액션아이템
- [ ] 할 일 — 담당 — 기한
```

---

## 2. 요약 노트 (Summary Note)

```markdown
# [원자료/회의/강의 제목] 요약

> **TL;DR:** [한 문장.]

---

## 핵심 포인트
- 포인트 1
- 포인트 2
- 포인트 3

## 상세 내용
### [소주제 A]
[설명]

### [소주제 B]
[설명]

## 액션아이템
- [ ] 할 일 1
- [ ] 할 일 2

## 참고 / 출처
- [링크 또는 출처]
```

---

## 3. 정리 / 구조화 문서 (Structured Note)

두서없는 메모·정보를 정돈할 때 사용.

```markdown
# [주제]

> **개요:** [무엇에 대한 정리인지 1~2문장.]

---

## [범주 1]
| 항목 | 설명 |
|------|------|
|  |  |

## [범주 2]
- 항목
- 항목

## 정리 / 결론
[핵심 한 단락.]
```

---

## 4. 기술문서 / 가이드 (How-to Guide)

```markdown
# [작업/도구] 가이드

> **이 문서는** [무엇을 할 수 있게 해주는지].

---

## 개요
[목적과 결과물.]

## 사전 준비
- 필요한 것 1
- 필요한 것 2

## 단계별 절차
1. **[단계명]** — [설명]
   ```bash
   # 명령어 예시
   ```
2. **[단계명]** — [설명]

## 트러블슈팅
| 증상 | 원인 | 해결 |
|------|------|------|
|  |  |  |

## 참고
- [링크]
```

---

## 5. README형 (Project README)

```markdown
# [프로젝트/도구 이름]

[한 줄 설명.]

---

## 개요
[무엇이고 왜 쓰는가.]

## 설치 / 시작하기
```bash
# 설치 명령
```

## 사용법
[기본 사용 예시.]

## 구성 / 구조
- `폴더/파일` — 설명

## 참고
- [링크]
```

## 범용 정리문서 (유형이 애매할 때 기본값)

```markdown
# [제목]

> **요약:** [한두 문장.]

---

## 1. [첫 번째 핵심 섹션]
[내용 — 표 또는 불릿]

## 2. [두 번째 핵심 섹션]
[내용]

## 3. 정리
- 핵심 1
- 핵심 2
```
