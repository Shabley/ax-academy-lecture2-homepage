# 기사 → A4 1페이지 컬러바 브리핑 DOCX (article-briefing-a4)

## 목적

고연심 소장(딥컴퍼스AI연구소)이 기사 자료를 던지면, **A4 세로 1페이지에 컬러바·표로 정리된 완성형 브리핑 DOCX**를 즉시 만든다. 설명·미리보기가 아니라 **실제 파일**을 납품한다. 30초 안에 핵심이 잡히는 두괄식이며, 강의·임원 보고에 바로 쓸 수 있다.

이 스킬은 앞단(기사 수집·핵심 추출)만 담당하고, 뒷단(1페이지 DOCX 생성·검증)은 검증된 **`a4-balance-docx` 엔진을 그대로 호출**한다. 레이아웃을 새로 짜지 않는다.

---

## 입력 형식 (혼합 허용)

- **웹 링크/URL** — 기사, 블로그, 리포트, 공식 발표 페이지. `web_fetch`로 본문 취득.
- **업로드 파일** — PDF·이미지(스크린샷)·DOCX·TXT·MD. 유형별로 읽는다(필요 시 file-reading / pdf-reading 스킬 참조).
- **붙여넣은 텍스트** — 기사 본문·발췌.
- **복수 자료** — 여러 링크·파일을 주면 하나의 통합 1페이지 브리핑으로 합친다.

## 출력 형식

- **A4 세로 1페이지 컬러바 브리핑 DOCX** (딥컴퍼스 브랜드 컬러·맑은 고딕). 예외 없음.
- 요청 시에만 채팅 요약본/마크다운/PDF 병행.

---

## 작업 흐름

### 1단계 — 기사 수집·읽기
- URL이면 `web_fetch`로 각각 본문 취득. 실패 시 사용자에게 텍스트 붙여넣기를 요청.
- 업로드 PDF/이미지는 해당 스킬로 읽는다. 이미지 기사(스크린샷)는 화면에서 직접 판독.
- 붙여넣은 텍스트는 그대로 분석.
- **저작권 준수**: 원문을 그대로 복제하지 않고 전부 자기 표현으로 재작성. 인용은 15단어 미만·출처당 1회 이내.

### 2단계 — 핵심 추출 (A4 1장 밀도에 맞춰 압축)
자료에서 다음을 뽑아 **짧은 문장/키워드 단위**로 정리한다.
- **핵심 결론 3줄** — 이 기사가 말하는 가장 중요한 것 (두괄식 최상단)
- **주요 사실·데이터** — 수치, 발표, 변화 지점 (표 행 3~5개)
- **배경·맥락** — 왜 지금 중요한가 (1~2줄)
- **시사점(So What)** — 강의·컨설팅·현장에 주는 의미 (2~3개)
- **강의/비즈니스 활용 포인트** — 소장의 강의·제안에 녹이는 법 (2~3개)

> A4 한 장 제약상 문장은 간결하게. 근거는 결론 아래에 붙이되, 넘치면 삭제가 아니라 축약한다.

### 3단계 — spec 변환
추출한 내용을 `a4-balance-docx`의 **spec 객체**로 매핑한다. 표준 매핑:

- **타이틀바** — 기사 주제 제목 + Gold 서브(출처·날짜·"AI 트렌드 브리핑" 등 kicker)
- **정보 박스(2~4칸)** — 출처 / 발행일 / 분야 / 브리핑 작성일 등 메타
- **섹션 반복** = [컬러 섹션바] + [표]
  1. `핵심 결론` (Navy 섹션바) — 배지형 리스트 3줄
  2. `주요 사실·데이터` (Navy) — 표 3~5행 (항목 / 내용)
  3. `배경·맥락` (Grey band) — 1~2줄 밴드
  4. `시사점 (So What)` (Orange 섹션바) — 배지형 2~3개
  5. `강의·비즈니스 활용` (Green 섹션바) — 배지형 2~3개
- **하단 박스(2칸)** — 브리핑 작성자(고연심 소장 강점 부각) / 문의(emas7788@naver.com)

색 용도는 `a4-balance-docx`의 컬러 시스템을 그대로 따른다(Navy 기본, Orange=시사점, Green=활용, Grey=배경, Gold=라벨).

### 4단계 — 생성 + 1페이지 검증

**경로 A (권장) — `a4-balance-docx` 엔진 호출.** 스크립트가 존재하면 이 경로를 쓴다.
```bash
mkdir -p /home/claude/work
ls /mnt/skills/user/a4-balance-docx/scripts/build_notice.js && \
cp /mnt/skills/user/a4-balance-docx/scripts/*.js /home/claude/work/
export NODE_PATH=$(npm root -g)
python3 -c "import PIL" || pip install Pillow --break-system-packages
```
```javascript
const { buildNotice } = require("./build_notice.js");
buildNotice({
  spec,
  outPath: "/mnt/user-data/outputs/[기사주제]_브리핑_고연심.docx",
  workDir: "/home/claude/work"
}).then(res => console.log(res)); // { pages:1, overflow:false }
```
`buildNotice`가 승인 폰트 프리셋(제목17/섹션10/본문9.5)으로 먼저 시도하고, 2페이지면 간격·폰트·여백 순으로 자동 압축해 1페이지에 수렴시킨다.

**경로 B (폴백) — 엔진 스크립트가 없으면 `docx` 공개 스킬로 직접 생성.**
먼저 `/mnt/skills/public/docx/SKILL.md`를 view한 뒤, 아래 고정값으로 A4 1페이지 문서를 만든다.
- 컬러: Navy `1E3A5F` / Dark Navy `0D1B2E` / Gold `C9A84C` / Orange `D2691E` / Green `2E7D52` / Grey `5B6470`
- 폰트: 맑은 고딕. 제목 17pt / 섹션바 10pt(흰색 굵게, 컬러 셀 배경) / 본문 9.5pt / 하단 8pt
- 여백(twip): top 620 / bottom 560 / left·right 900. 섹션 간격 최소화.
- 레이아웃: 타이틀바(Dark Navy 셀) → 정보박스(2~4칸 표) → 섹션바+표 반복 → 하단 2칸 박스.
- 생성 후 **PDF로 변환해 페이지 수를 확인**(`libreoffice --headless --convert-to pdf`)하고, 2페이지면 섹션 간격 → 셀 패딩 → 폰트 1pt → 여백 순으로 줄여 재검증한다. 내용 삭제가 아니라 축약으로만 맞춘다.

### 5단계 — 납품
- `res.pages === 1` → `present_files`로 납품.
- `res.overflow === true` → 최종 압축에도 미수렴. 2단계 추출 문장을 **더 간결히 축약**(삭제 아님)한 뒤 재실행.

- 경어체(~합니다/~입니다).
- 복수 기사는 상충 지점을 명시하고 통합 관점을 제시.
- 별도 지정 없으면 작성자=고연심·딥컴퍼스AI연구소, 문의=emas7788@naver.com.
- 하단 작성자 박스에 소장 강점(문화예술학 박사·연구소 소장·20년+ 신기술 강의) 항상 부각.
